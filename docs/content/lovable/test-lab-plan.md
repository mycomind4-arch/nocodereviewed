# Lovable Test Lab Plan

## SaaS dashboard

**Starter prompt:** Build a subscription analytics dashboard with auth, organizations, user roles, revenue charts, customer table, billing settings, and admin-only controls.

**What to measure:** Measure prompt-to-app completeness, schema quality, protected routes, dashboard state, role separation, and deploy behavior.

**Evidence to capture:** prompt, screenshots, generated schema, generated code notes, deploy result, errors, security findings, production-readiness notes.

**Status:** Planned / not completed.

## Client portal

**Starter prompt:** Build a client portal where clients can log in, view project status, upload files, message the team, approve deliverables, and see invoices.

**What to measure:** Measure auth flows, file permissions, client isolation, admin access, notifications, and data privacy.

**Evidence to capture:** prompt, screenshots, generated schema, generated code notes, deploy result, errors, security findings, production-readiness notes.

**Status:** Planned / not completed.

## Marketplace MVP

**Starter prompt:** Build a local services marketplace with vendor profiles, listings, search filters, booking requests, reviews, and admin moderation.

**What to measure:** Measure multi-role complexity, vendor/customer permissions, moderation, form validation, and workflow durability.

**Evidence to capture:** prompt, screenshots, generated schema, generated code notes, deploy result, errors, security findings, production-readiness notes.

**Status:** Planned / not completed.

## Internal CRM

**Starter prompt:** Build an internal CRM with lead stages, task assignments, notes, tags, saved filters, activity history, and manager-only reporting.

**What to measure:** Measure CRUD reliability, relational data model, permissions, auditability, search/filter usability, and realistic team workflow.

**Evidence to capture:** prompt, screenshots, generated schema, generated code notes, deploy result, errors, security findings, production-readiness notes.

**Status:** Planned / not completed.

## AI content tool

**Starter prompt:** Build an AI content planner with project folders, saved prompts, generated drafts, approval status, export options, and API-key-safe AI integration.

**What to measure:** Measure API key handling, prompt workflow quality, data persistence, revision history, and safe AI feature configuration.

**Evidence to capture:** prompt, screenshots, generated schema, generated code notes, deploy result, errors, security findings, production-readiness notes.

**Status:** Planned / not completed.
