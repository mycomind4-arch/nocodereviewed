# VibeCode Authority Competitive Intelligence Model

## Principle

The site should not treat every AI builder as the same kind of product. Rankings become credible only when tools are compared inside the correct market segment, buyer intent, and benchmark family.

## Market Segments

| Segment | Role | Representative tools | Primary test |
| --- | --- | --- | --- |
| Prompt-to-app leaders | Core category | Lovable, Bolt.new, Base44, Tempo, Create | SaaS MVP benchmark, deployment path, data model handling, export/handoff |
| Agentic IDEs | Technical escalation path | Cursor, Windsurf, Replit Agent, Firebase Studio | Existing-codebase benchmark, repo edits, debugging, maintainability |
| UI and design generation | Front-end acceleration | v0, Uizard, Galileo AI, Builder.io Visual Copilot | UI fidelity, component quality, responsiveness, integration readiness |
| No-code incumbents | Structured alternatives | Bubble AI, FlutterFlow AI, Webflow AI, Framer AI | Control versus speed, lock-in, scalability, data ownership, AI assist depth |
| Internal tool builders | Business operations lane | Retool AI, Softr AI, Appsmith AI | Permissions, connectors, auditability, database safety, workflow reliability |

## Competitive Signals

| Signal | Impact | Response |
| --- | --- | --- |
| Pricing or credit model change | Shifts value recommendations and affiliate CTA language | Refresh pricing snapshots, value claims, comparison tables, and best-of rankings |
| New deployment, export, or database feature | Can move a tool from prototype-safe to pilot-safe or production-candidate | Rerun production-readiness checks and update affected verdicts |
| Major model or agent workflow upgrade | Changes benchmark speed, repair difficulty, architecture, and UI quality | Rerun benchmark prompts and compare against prior artifacts |
| Security, privacy, or terms change | Can block enterprise, sensitive-data, or production recommendations | Update policy snapshots and require human review before claims publish |
| New entrant gains category attention | May require review coverage and benchmark inclusion | Add to intake queue, classify segment, and assign first benchmark task |

## Watch Priorities

- Keep Lovable, Bolt.new, Replit Agent, v0, Cursor, Base44, and Windsurf as the first authority cluster.
- Separate prompt-to-app builders from AI IDEs so comparisons do not flatten different use cases.
- Track no-code incumbents as alternatives, not as identical vibe-coding products.
- Promote a tool in rankings only after fresh benchmark evidence supports the move.
- Treat sudden category buzz as an intake trigger, not as proof of quality.

## Operating Workflow

1. Classify each tool into a market segment before writing a verdict.
2. Assign a benchmark family that matches the segment.
3. Track pricing, feature, model, policy, and category-buzz signals.
4. Trigger source refreshes and benchmark reruns when a signal affects claims.
5. Update reviews, comparisons, best-of pages, reports, and social assets only after evidence changes.

## Outcome

This model prevents sloppy comparisons. VibeCode Authority can say which tool is best for a use case because it first defines the category, the buyer, the benchmark, and the evidence threshold.
