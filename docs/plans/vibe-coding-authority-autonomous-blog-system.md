## Editorial Moat

The site should not win by publishing more posts. It should win by publishing posts competitors cannot easily copy.

Every review should include:

- The exact benchmark prompt used.
- The app idea tested.
- Screenshots or output links.
- Setup friction notes.
- Auth/database/deployment findings.
- Pricing and credit usage notes.
- Export/handoff assessment.
- Security and production-readiness notes.
- Final verdict by use case.

If a post lacks evidence, it does not publish.

## Autonomous Pipeline

### Layer 1: Source Watcher

Monitors:

- Official docs.
- Pricing pages.
- Changelogs.
- Release notes.
- Product blogs.
- GitHub repos where relevant.
- Public status pages.
- Affiliate program pages.

Outputs:

- Tool change events.
- Pricing change events.
- New feature events.
- Broken/deprecated claim alerts.

### Layer 2: Benchmark Runner

Runs standardized tests across the tool set.

Benchmark families:

- SaaS MVP: auth, dashboard, billing placeholder, database table.
- Directory app: listings, filters, detail pages, submit form.
- Internal tool: CRUD dashboard, role fields, CSV import.
- Landing page: responsive page, form capture, pricing section.
- AI wrapper: prompt form, API call placeholder, result history.

Each run records:

- Prompt.
- Tool.
- Date.
- Output URL or files.
- Build time.
- Error count.
- Manual fixes required.
- Deploy success.
- Security issues.
- Reviewer notes.

### Layer 3: Evidence Store

Required tables:

- `tools`
- `tool_pricing_snapshots`
- `tool_feature_snapshots`
- `benchmark_runs`
- `benchmark_artifacts`
- `review_evidence`
- `claim_registry`
- `article_sources`
- `article_quality_scores`
- `refresh_events`

The article generator can only make claims that map back to evidence or an approved source.

### Layer 4: Article Generator

Allowed article types:

- Tool review.
- Tool vs tool comparison.
- Best tool for use case.
- Benchmark report.
- Pricing change report.
- Production readiness guide.
- Build log.
- Migration/handoff guide.

Generation inputs:

- Tool facts.
- Benchmark runs.
- Pricing snapshots.
- Evidence notes.
- Approved editorial template.
- Internal link map.
- Affiliate disclosure rules.

Generation is not allowed to invent:

- Pricing.
- Feature support.
- Security claims.
- Performance claims.
- Production readiness.
- Affiliate terms.

### Layer 5: Quality Gateway

Hard rejects:

- No evidence records.
- No clear verdict.
- No last-verified date.
- No affiliate disclosure when affiliate links exist.
- Pricing claim older than 7 days.
- Feature claim with no source or benchmark evidence.
- Generic listicle phrasing.
- Unsupported "best" claim.
- No internal links.
- No benchmark or build-log insight.

Scored dimensions:

- Specificity.
- Evidence density.
- Verdict clarity.
- Production usefulness.
- Pricing accuracy.
- Tool-fit nuance.
- Original insight.
- Security awareness.
- Reader decision value.
- Internal linking.

### Layer 6: Autonomous Refresh Agent

Refresh triggers:

- Pricing changed.
- Tool shipped major feature.
- Tool docs changed.
- Article impressions rose but CTR is low.
- Affiliate click rate changed.
- Benchmark evidence expired.
- Competing tool overtook ranking.
- Product became unavailable.

Refresh outcomes:

- Update facts only.
- Add new evidence block.
- Rewrite verdict.
- Re-run benchmark.
- Deprecate article.
- Split article into deeper comparison.

## Publishing Schedule

Early stage:

- 2 benchmark-backed reviews per week.
- 1 comparison per week.
- 1 build log per week.
- 1 pricing/market update per week.

Do not publish daily until:

- 20 evidence-backed reviews exist.
- The Quality Gateway pass rate is stable.
- Refresh automation is working.
- At least 10 benchmark runs are repeatable.

Mature stage:

- 1 article per weekday.
- 1 benchmark report per week.
- 1 tool update digest per week.
- 1 major comparison refresh per week.
