# VibeCode Authority Design System Model

## Principle

The brand has two jobs: create authority and preserve evidence clarity. Cinematic institutional visuals should build memorability, while the product interface should stay fast, readable, and proof-first.

## Design Layers

| Layer | Use | Tone | Avoid |
| --- | --- | --- | --- |
| Authority layer | Heroes, report covers, social cards, launch announcements, YouTube banners, and institutional brand moments | Dark, cinematic, precise, infrastructure-grade, high-contrast | Fake product evidence, unreadable UI, generic AI glow, or visuals that imply unsupported proof |
| Evidence layer | Reviews, comparison tables, benchmark pages, citations, QA states, admin capture, and data-heavy workflows | Light, fast, readable, dense, restrained, operational | Marketing hero bloat, vague badges, unsupported winner emphasis |
| Report layer | Platform indexes, monthly reports, executive summaries, PDFs, stakeholder findings | Institutional, dated, sober, data-forward | Undated rankings, sponsor confusion, decorative charts, hidden methodology |
| Operational layer | Admin, readiness, automation, evidence graph, correction workflow, source watcher screens | Compact, scannable, action-oriented, low ornament | Ambiguous status colors, hidden blockers, oversized headings inside dense tools |

## Component Standards

| Component | Requirement |
| --- | --- |
| Verdict module | Shows score, provisional/final state, evidence badges, disclosure note, and last source check |
| Review card | Shows category, fit, score, gate state, and one next action |
| Comparison table | Keeps both tools visible, marks evidence gaps, and avoids winner language until parity exists |
| Benchmark card | Shows prompt family, task, deploy result, artifact status, and manual repair count when available |
| Trust badge | Uses controlled badge language and maps to a real evidence state |
| Report cover | Shows title, issue date/evidence window, methodology link, and sponsor state when relevant |

## Interface Rules

- Evidence interfaces should prioritize tables, grids, compact cards, badges, and clear blocked states.
- Authority visuals should not be used as proof unless they are covers or promotional assets pointing back to evidence.
- Dense panels need stable layout dimensions so badges, labels, and dynamic counts do not cause major shifts.
- Reader-facing CTAs must sit after evidence or disclosure context.
- Use restrained contrast and typography for data surfaces; reserve cinematic contrast for authority assets.

## Operating Outcome

This model keeps the ecosystem from splitting into unrelated aesthetics. VibeCode Authority can look institutional in public media while remaining readable, usable, and evidence-first inside reviews and benchmark pages.
