# VibeCode Authority Visual Asset Production Model

## Principle

Visuals should strengthen authority without pretending to be evidence. The cinematic brand layer creates memorability. The evidence layer shows what was tested, when it was tested, and what is still provisional.

## Asset Queue

| Asset | Layer | Use | Prompt Need | Proof Boundary |
| --- | --- | --- | --- | --- |
| Homepage Authority Hero | Authority | First-viewport brand signal and launch social crop | Dark institutional benchmark command center with VCA identity, no fake product UI | Brand atmosphere only; cannot imply benchmark evidence |
| Review Lead Images | Evidence | Top review visuals and social cards for core platform reviews | Clean interface composition with tool category, evidence badges, and readable verdict state | Must not use unofficial logos unless rights and source assets are approved |
| Benchmark Result Cards | Evidence | Shareable result images for same-prompt tool comparisons | Data-first card with prompt name, date, score, blockers, and artifact state | Only generated after benchmark run and QA state exist |
| Monthly Report Covers | Authority | Platform Index, benchmark reports, and production watchlists | Institutional cover with evidence window, report title, VCA mark, and category map | Cover cannot show rankings that are not present in the report data |
| YouTube and Newsletter Frames | Hybrid | Video thumbnails, newsletter headers, digest sections, and launch recaps | Authority frame plus one evidence module: score movement, ranking shift, or blocker | Headline must match published page and disclosure status |

## Prompt Standards

| Standard | Include | Avoid |
| --- | --- | --- |
| Authority Layer | VCA mark, institutional architecture, benchmark infrastructure, dated report context | Generic AI glow, fake code screens, unreadable tiny UI, unsupported ranking claims |
| Evidence Layer | Readable cards, dates, source state, benchmark state, disclosure state, artifact status | Cinematic lighting that hides data or makes generated screenshots look like proof |
| Platform-Specific Assets | Category label, use case, verdict state, and canonical page reference | Unofficial product logo recreation, trademark confusion, or fabricated screenshots |
| Distribution Crop Set | 16:9, 1:1, 4:5, OG, newsletter header, and YouTube-safe variants | Text placed near crop edges or layouts that fail on mobile feeds |

## Production Workflow

1. Choose the asset surface and layer.
2. Pull the canonical page, report, or benchmark record.
3. Confirm the verdict state, evidence state, disclosure state, and source date.
4. Generate or design the visual.
5. Verify crops for homepage, OG, social, newsletter, and video use.
6. Store prompt, source record IDs, asset version, and proof boundary.

## Operating Rule

Authority visuals can make the brand memorable. Evidence visuals must make the proof inspectable.
