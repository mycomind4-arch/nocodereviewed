# VibeCode Authority Publishing Calendar Model

## Principle

The publishing calendar exists to sequence evidence, not just articles. A page earns a launch slot only when its benchmark work, source snapshots, artifacts, internal links, distribution asset, and QA path are clear.

## Cadence

| Rhythm | Output | Owner | Rule |
| --- | --- | --- | --- |
| Daily | Evidence capture | Research agent | Save source snapshots, artifacts, benchmark notes, and blockers before drafting advances |
| Twice weekly | Core reviews or comparisons | Editorial agent | Publish only when benchmark evidence, fresh source data, artifacts, and QA pass exist |
| Weekly | Digest and update log | Distribution agent | Summarize changed evidence, score movement, blockers, and publish-ready reviews |
| Monthly | Authority report | Report agent | Use a dated evidence window with methodology, exclusions, and changed verdicts |

## Four-Week Launch Sequence

### Week 1: Foundation Reviews

Ship:
- Lovable Review
- Bolt.new Review
- Replit Agent Review

Required evidence:
- SaaS MVP benchmark
- Pricing snapshot
- Deploy result
- Artifact
- Final QA

Blocker:
- Do not rank best overall until the same prompt has run across all three tools.

### Week 2: Comparison Cluster

Ship:
- Lovable vs Bolt.new
- Lovable vs Replit Agent
- Bolt.new vs Replit Agent

Required evidence:
- Side-by-side benchmark artifacts
- Same scoring model
- Source freshness under seven days

Blocker:
- Comparison verdicts must disclose incomplete benchmark or production gates.

### Week 3: IDE And Codebase Tools

Ship:
- Cursor Review
- Windsurf Review
- Cursor vs Windsurf

Required evidence:
- Existing-codebase benchmark
- Handoff notes
- Repo inspection
- Pricing and terms snapshots

Blocker:
- Do not compare IDEs against prompt-to-app tools without explicit use-case boundary language.

### Week 4: Authority Packaging

Ship:
- Best Vibe Coding Tools
- Platform Index
- Weekly benchmark digest

Required evidence:
- Completed evidence gates for ranked tools
- Claim ledger
- Report evidence window

Blocker:
- Best-of pages must mark provisional entries until full evidence exists.

## Publication Gates

- Every scheduled page needs a target keyword, evidence gate, internal links, and distribution asset.
- A page moves from queued to drafting only after required source snapshots and benchmark tasks are identified.
- A page moves from drafting to review only after artifacts and review records exist.
- A page publishes only after QA passes factuality, evidence, originality, SEO, and disclosure checks.
- A page refreshes when pricing, limits, positioning, or benchmark evidence changes.

## Operating Notes

- Reviews create the evidence base.
- Comparisons convert high-intent search demand.
- Best-of pages aggregate evidence after enough reviews are complete.
- Reports package the system into a recurring authority product.
- Digests keep the audience aware of evidence changes without inventing new claims.
