# VibeCode Authority Content Operating Model

## Purpose

VibeCode Authority should publish like a benchmark institution, not a generic review blog. Every review, comparison, best-of page, report, and build log must be traceable to saved evidence.

## Publishing Lanes

| Lane | Priority | Output | Publish Gate |
| --- | --- | --- | --- |
| Launch reviews | P0 | Core reviews for Lovable, Bolt.new, Replit Agent, v0, Cursor, Base44, Windsurf | Review record, benchmark run, fresh source snapshot, artifact, final QA |
| Comparison pages | P0 | Versus pages and alternatives pages | Same prompt run across compared tools, comparable artifacts, fresh snapshots |
| Best-of hubs | P1 | Use-case ranking pages | At least three ranked tools with complete evidence or clear provisional labels |
| Authority reports | P1 | Recurring benchmark products | Report-specific evidence checklist, claim ledger compliance, dated evidence window |
| Build logs | P2 | Original proof assets | Prompt transcript, screenshots, app URL or export, fixes, deploy result, score |

## Claim Standards

Strong claims are blocked unless the required proof exists.

| Claim | Required Evidence | Fail Condition |
| --- | --- | --- |
| Best overall | Same benchmark family across direct competitors plus fresh source snapshots | Use provisional language until all compared tools pass |
| Production-ready | Deploy result, auth/data review, secrets handling, export path, QA notes | Replace with prototype-ready if any gate is missing |
| Cheapest or best value | Current pricing snapshot, credit limits, usage constraints, scaling notes | Block if pricing evidence is older than seven days |
| Fastest | Timed build log, prompt count, manual repair count, final app status | Do not claim speed from marketing copy |
| Best for nontechnical founders | Observed workflow, app completeness, repair difficulty, handoff notes | Qualify if code or terminal work is required |

## Operating Rule

The editorial system should always ask: what evidence is missing before this can publish?

The next action is determined by the first missing gate:

1. Review record
2. Benchmark run
3. Fresh official source snapshot
4. Artifact
5. Final QA

If any gate is missing, the article can exist as a draft or provisional page, but it cannot present final recommendations.
